# AI 裁判 P0 PRD 对齐验收报告（2026-04-12）

## 范围

本轮按 `AI_Judge_Service PRD 对齐 P0 开发计划（决策完成版）` 执行，闭环以下 4 个 P0 缺口：

1. final 展示字段硬切（`debateSummary / sideAnalysis / verdictReason`）
2. trace + replay 闭环（新增 `POST /internal/judge/jobs/{job_id}/replay`）

4. 输入盲化拒绝（`extra=forbid` + 敏感键兜底校验）

## 变更落地

### 1) ai_judge_service

- final 合同校验硬切为 `debateSummary/sideAnalysis/verdictReason` 必填。
- dispatch 全流程接入 `trace_store.register_start/register_success/register_failure`。
- 新增 `POST /internal/judge/jobs/{job_id}/replay`，支持 `dispatch_type=auto|final|phase`，无 callback 副作用，写入 `mark_replay` 历史。
- phase/final 回调客户端扩展为四通道：`report + failed`。
- phase/final 失败、final 合同阻断、盲化拒绝均触发 failed callback。
- request/message 模型启用 `extra=forbid`，并增加敏感键扫描（命中返回 `422 input_not_blinded`）。

### 2) chat_server

- 新增内部回调路由：
  - `POST /api/internal/ai/judge/v3/phase/jobs/{id}/failed`
  - `POST /api/internal/ai/judge/v3/final/jobs/{id}/failed`
- 新增 failed callback 输入/输出类型并接入 OpenAPI。
- final report 入参与落库硬切：
  - 删除主语义 `final_rationale` 依赖路径
  - 使用 `debate_summary / side_analysis / verdict_reason`
- 查询与映射链路同步更新（rows/helpers/query/tests）。
- 新增迁移：
  - `chat/migrations/20260412003000_ai_judge_final_display_contract_cutover.sql`
