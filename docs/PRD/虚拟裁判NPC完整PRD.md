# 虚拟裁判 NPC 完整 PRD（v0.2）

更新时间：2026-05-01  
文档状态：新功能 PRD 草案  
适用范围：Debate Room 内直接可见、可与用户互动的娱乐化虚拟裁判 NPC 产品能力

---

## 0. 文档目的

本文档用于定义 EchoIsle 的“虚拟裁判 NPC”功能，明确：

1. 虚拟裁判 NPC 在辩论房间中的娱乐化定位、用户价值与边界。
2. 用户能看到什么、能与 NPC 发生什么互动、NPC 能主动做什么。
3. NPC 行为、房间秩序、趣味表现、权限边界与验收口径。
4. 与赛后 AI 裁判团、正式裁决报告、运营管理能力之间的关系。

说明：

1. 本文档只描述产品功能细节，不展开模型选择、Agent 编排、服务架构、工具调用协议或技术实现。
2. 虚拟裁判 NPC 是赛中可见、娱乐导向的公开互动角色，不替代赛后官方 AI 裁判团。
3. 正式公开裁决仍以平台终局裁判流程为准，虚拟裁判 NPC 的赛中行为不得直接等同于最终胜负判定。
4. 虚拟裁判 NPC 不提供用户私聊入口；用户只能在房间公开语境中与其发生互动。

---

## 1. 功能定位

### 1.1 功能定义

虚拟裁判 NPC 是 Debate Room 中常驻、公开可见、娱乐导向、可主动观察辩论并与房间内用户互动的 AI 角色。

它在用户感知上类似一名“现场裁判 / 主持人 / 气氛调度者 / 秩序维护者”：

1. 能以动态形象常驻在辩论房间内，让用户明确感知到“现场裁判在场”。
2. 能看到当前辩论上下文与房间公开事件。
3. 能以 NPC 身份在房间中公开发言。
4. 能对精彩发言、关键反驳、幽默表达或高能交锋做出赞赏和气氛反馈。
5. 能在规则允许范围内执行特定房间动作。
6. 能根据辩论进程主动决定是否提示、提醒、暂停、强调重点或触发视觉效果。
7. 能让用户明确区分赛中娱乐化 NPC 与赛后官方裁决，而不是把它误认为最终判决来源。

### 1.2 与 AI 裁判团的区别

| 维度 | 虚拟裁判 NPC | AI 裁判团 |
|---|---|---|
| 用户可见性 | 赛中公开可见 | 主要在赛后通过裁决报告可见 |
| 主要目标 | 娱乐互动、气氛、节奏、秩序、可感知陪伴 | 正式判决、评分、报告、复核 |
| 行为时机 | 辩论进行中持续存在 | 场次进入可判决状态后触发 |
| 输出形式 | 动态形象反馈、房间发言、赞赏、提醒、暂停、效果、阶段提示 | 官方结果、评分、理由、证据链 |
| 权威程度 | 现场辅助与规则执行 | 官方终局裁决 |
| 是否决定胜负 | 不直接决定最终胜负 | 决定 `pro / con / draw` 或复核状态 |

### 1.3 核心价值

1. 增强辩论过程中的 AI 存在感，让用户不是只在赛后才感知到 AI。
2. 通过赞赏、吐槽式提醒、气氛反馈和阶段仪式感，让辩论房间更像一场正在发生的娱乐赛事。
3. 提升房间秩序和节奏感，减少跑题、刷屏、等待、冷场等体验问题。
4. 为辩论增加娱乐性、仪式感和观赏性，提高用户停留和观战体验。
5. 帮助用户理解当前阶段、规则、争点变化与后续动作。
6. 为未来扩展“不同风格裁判”“赛事主持”“付费房间特效”等能力提供产品基础。

## 2. 用户角色

### 2.1 参赛用户

参赛用户是正方或反方成员，主要需求包括：

1. 明确知道当前辩论阶段、剩余时间与规则边界。
2. 在辩论混乱时获得秩序维护。
3. 看到裁判对关键争点、跑题、重复发言或不当行为的即时提示。
4. 在精彩发言被公开赞赏时获得正反馈和参与成就感。
5. 感受到辩论像一场有现场气氛的娱乐赛事，而不是普通聊天室。

### 2.2 观战用户

观战用户不参与阵营发言，主要需求包括：

1. 快速理解当前辩论进行到哪里。
2. 看懂双方正在争什么。
3. 通过 NPC 的动态形象、赞赏、提示和效果增强观赛乐趣。
4. 在关键时刻知道为什么房间发生暂停、警告、阶段切换或特殊效果。
5. 通过 NPC 的气氛调动更容易感知“这一刻很精彩”。

### 2.3 运营用户

运营用户负责内容、房间规则与赛事体验，主要需求包括：

1. 配置 NPC 的默认风格、可用动作和触发边界。
2. 关闭或限制不适合当前活动的 NPC 能力。
3. 查看 NPC 行为是否对房间体验产生负面影响。
4. 在必要时人工接管或撤销 NPC 的部分现场行为。

## 3. 产品目标

### 3.1 核心目标

1. 让每个启用 NPC 的房间都有清晰可感知、动态可见的“娱乐化现场裁判”存在。
2. 让 NPC 的行为服务于辩论气氛和观赛体验，而不是抢夺用户表达空间。
3. 在不替代正式裁决的前提下，提升赛中秩序、节奏、趣味和可理解性。
4. 让用户能够自然区分“NPC 现场娱乐互动”和“赛后官方裁决”。

### 3.2 成功指标

1. NPC 启用房间的平均停留时长提升。
2. 观战用户对关键阶段的点击、停留或互动率提升。
3. 精彩发言被赞赏后的用户互动、继续发言或观战反馈提升。
4. 房间跑题、刷屏、重复提醒、人工干预次数下降。
5. 用户对 NPC 发言的负反馈率低于可接受阈值。
6. 参赛用户认为 NPC “打断过多 / 偏袒一方 / 影响发挥”的投诉率保持在低位。

## 4. 功能范围

### 4.1 首版包含

1. 房间内展示一个明确的虚拟裁判 NPC 身份。
2. NPC 在房间内以动态形象出现，可通过姿态、表情、动作或动效反馈参与感。
3. NPC 可公开发送裁判消息。
4. NPC 可对精彩发言、高质量反驳、幽默表达或关键转折进行赞赏。
5. NPC 可进行阶段提示、规则提醒、争点提示、秩序提醒。
6. NPC 可在规则允许下触发暂停、恢复、警告、特效等房间行为。
7. 用户可看到 NPC 的行为原因或简短说明。
8. 运营可控制 NPC 是否启用、风格强度和关键能力开关。

### 4.2 永久边界

以下能力不只是首版不做，而是该功能的长期产品边界：

1. NPC 永远不会替代 AI 裁判团生成正式裁决报告。
2. NPC 永远不会作为最终胜负的官方判定来源。
3. 用户不能私聊 NPC，也不能通过 NPC 获取个人专属辩论建议。
4. NPC 不代替用户发言、自动参赛或自动站队。
5. NPC 不向用户暴露 AI 裁判团内部推理、评分、置信度或审计字段。
6. NPC 不因赞赏某个用户的精彩发言而改变正式裁决标准。

### 4.3 阶段外能力

以下能力首版不做，但未来可独立评估：

1. 复杂多 NPC 角色剧场。
2. 用户自定义 NPC 皮肤、语气或展示风格。
3. 用户自定义 NPC 模型、Prompt 或完整行为脚本。

## 5. 核心体验原则

### 5.1 可见但不喧宾夺主

NPC 必须让用户感知到它在场，但不能变成房间的主角。

产品要求：

1. NPC 消息在视觉上与用户消息区分。
2. NPC 不应高频刷屏。
3. NPC 重要动作需要有明确视觉反馈。
4. NPC 普通提示应轻量，不打断用户正在进行的表达。

### 5.2 公平中立但允许赞赏

NPC 在公开房间内必须保持中立，不能表现出对正方或反方的无依据偏袒。由于 NPC 是娱乐导向角色，它可以自由决定赞赏某个用户的公开发言，但赞赏对象必须基于发言内容本身，而不是用户身份或阵营倾向。

产品要求：

1. 对双方使用一致的称谓、语气和提醒标准。
2. 不提前暗示某一方会赢。
3. 不公开给单个用户贴负面标签。
4. 可以点名赞赏某个用户的精彩表达、关键反驳、幽默表达或有效追问。
5. 赞赏应解释“这段发言精彩在哪里”，避免被理解为“这个用户或阵营一定领先”。
6. 争点提示应围绕内容与规则，不围绕用户身份、消费能力或历史表现。

### 5.3 行为可解释

当 NPC 执行影响房间节奏的动作时，用户应知道发生了什么。

产品要求：

1. 暂停、恢复、警告、禁言建议等强动作必须附带简短原因。
2. 特效类动作应让用户理解它对应的是“关键转折”“高能发言”“阶段切换”等体验事件。
3. 若 NPC 行为被人工撤销，应在必要时给出房间内可见说明。

### 5.4 正式裁决隔离

NPC 的赛中提示不得与赛后正式裁决混淆。

产品要求：

1. NPC 可提示“当前争点发生变化”，但不能宣布“本场已经由某方获胜”。
2. NPC 可提醒某方需要回应某问题，但不能公开给出阵营评分。
3. NPC 可做阶段性总结，但必须避免被用户理解为最终判决。
4. NPC 可赞赏某个用户的单条发言，但赞赏不构成官方评分、胜负倾向或裁决证据。

### 5.5 娱乐导向

NPC 的默认气质应偏娱乐、活跃和有现场感。

产品要求：

1. NPC 可以通过动态形象、短句、赞赏、轻量调侃和特效活跃房间气氛。
2. NPC 的娱乐表达应围绕公开发言、比赛节奏和现场氛围，不围绕用户隐私或身份标签。
3. NPC 可以制造“高能时刻”“漂亮反击”“关键追问”等观赛信号。
4. NPC 的娱乐表达不能压过辩论主体，也不能让用户误解为正式裁决。

## 6. NPC 展示需求

### 6.1 房间身份展示

启用 NPC 的房间中，应展示一个固定的裁判身份入口，并在房间主体验中呈现动态角色形象。

至少包含：

1. NPC 名称。
2. NPC 动态形象或角色展示区。
3. 当前状态：观察中、发言中、赞赏中、暂停中、人工接管中、不可用。
4. 简短身份说明：例如“虚拟裁判”“现场主持”“气氛裁判”“规则提醒”。
5. 与正式裁决入口明显不同的视觉风格。

### 6.2 动态形象展示

动态形象是 NPC 的核心用户感知之一，具体视觉方案由前端设计定义，但产品上应满足：

1. 用户进入房间后能立即知道 NPC 是一个赛中动态角色，而不是赛后裁决报告。
2. NPC 在观察、发言、赞赏、暂停、触发特效等状态下有可感知的动态反馈。
3. 动态反馈不遮挡聊天、发言输入、倒计时和关键操作。
4. 用户在低性能设备或不喜欢动效时，可以关闭或降低动态表现。
5. 动态形象应更偏娱乐现场感，而不是权威审判感，避免让用户误以为它正在生成正式判决。

### 6.3 消息样式

NPC 消息应区别于普通用户消息。

至少支持：

1. 普通提示消息。
2. 重要裁判消息。
3. 阶段切换消息。
4. 暂停 / 恢复消息。
5. 赞赏消息。
6. 特效触发消息。

### 6.4 状态反馈

用户应能感知 NPC 当前是否正在参与房间。

典型状态：

1. 观察中：NPC 正在关注辩论，但当前不发言。
2. 发言中：NPC 正在发布提示或总结。
3. 赞赏中：NPC 正在对某段公开发言做正向反馈。
4. 执行动作中：NPC 正在触发暂停、恢复或特效。
5. 静默中：NPC 被系统或运营限制，不主动发言。
6. 人工接管中：运营或管理员暂时控制 NPC 行为。
7. 不可用：NPC 功能异常或当前房间未启用。

## 7. NPC 行为能力

### 7.1 裁判发言

NPC 可以在房间内发送公开消息。

发言类型包括：

1. 开场提示：说明辩题、双方立场、基本规则。
2. 阶段提示：提醒当前进入开场、自由辩、总结、等待判决等阶段。
3. 规则提醒：提示发言规范、禁止刷屏、不得人身攻击等。
4. 争点提示：指出当前双方正在围绕哪些问题交锋。
5. 回应提醒：提醒某方仍有关键问题未回应。
6. 节奏提示：提示发言过密、过慢、重复或偏离主题。
7. 高能提示：标记一段明显推动辩论进展的发言。
8. 赞赏提示：夸赞某个用户的精彩发言、关键反击或有效追问。
9. 收束提示：在阶段末尾总结当前主要争点，不给最终胜负判断。

### 7.2 赞赏与气氛调动

NPC 可以主动赞赏某个用户的公开发言，帮助制造辩论现场的正反馈和观赛高能感。

可赞赏场景：

1. 用户提出清晰、有力、推动争点前进的论证。
2. 用户对对方核心问题做出有效回应。
3. 用户提出精准追问，让争点更清楚。
4. 用户用幽默但不攻击的表达活跃气氛。
5. 用户总结出让观战者更容易理解的关键分歧。

赞赏内容要求：

1. 可以点名用户或引用其公开发言片段。
2. 应说明被赞赏的是哪种表达质量，例如“反击清楚”“追问有效”“例子有帮助”。
3. 不得把赞赏升级为阵营优势、胜负预测或官方评分。
4. 对双方阵营都应保持开放，不要求机械平均，但长期表现不能显著偏向某一阵营。
5. 赞赏语气可以更活泼，但不能羞辱未被赞赏的用户。

### 7.3 暂停与恢复

NPC 可以在符合规则的情况下暂停辩论。

可触发暂停的产品场景：

1. 房间出现严重刷屏或重复无意义内容。
2. 双方争议集中在规则理解，需要临时澄清。
3. 出现明显不当内容，需要给用户提示和整理时间。
4. 运营配置的赛事流程要求阶段性暂停。
5. 系统需要短暂同步房间状态。

暂停期间：

1. 页面应明确展示当前为暂停状态。
2. 用户应看到暂停原因。
3. 若允许继续发送消息，应区分“正式辩论发言”和“暂停期间讨论”。
4. 若不允许继续发送消息，应给出明确提示。

恢复时：

1. NPC 应发布恢复说明。
2. 页面应明确恢复正常辩论状态。
3. 若暂停影响倒计时或阶段，应展示恢复后的时间状态。

### 7.4 警告与秩序维护

NPC 可以对房间行为进行公开警告或提醒。

警告等级建议：

1. 轻提醒：语气轻，提示回到主题或减少重复。
2. 明确警告：指出行为已经影响辩论秩序。
3. 强制动作建议：提示管理员或系统可能采取限制措施。

NPC 不应直接做长期处罚决定，除非该动作已经被房间规则明确授权。

### 7.5 特效与仪式感

NPC 可以触发房间内视觉、音效或动态形象反馈，用于增强观赛体验。

可用特效类型：

1. 开场仪式特效。
2. 阶段切换特效。
3. 关键争点出现特效。
4. 高能发言特效。
5. 精彩发言赞赏特效。
6. 幽默表达或气氛回暖特效。
7. 平局投票开启提示特效。
8. 等待正式裁决提示特效。

特效要求：

1. 不遮挡核心聊天内容过久。
2. 不影响用户发言操作。
3. 对移动端和桌面端都应保持可读。
4. 用户应能关闭或降低动效强度。

### 7.6 阶段总结

NPC 可以在辩论过程中生成公开阶段总结。

阶段总结应包含：

1. 当前阶段内双方主要论点。
2. 尚未被充分回应的关键问题。
3. 争点是否发生转移。
4. 下一阶段建议关注的讨论方向。

阶段总结不得包含：

1. 最终胜负判断。
2. 公开阵营评分。
3. 对单个用户的羞辱性评价。
4. 不可见内部裁判依据。

## 8. 用户交互

### 8.1 用户对 NPC 的公开呼叫

用户可以在房间内公开呼叫 NPC，但不能私聊 NPC。

常见呼叫场景：

1. 请求解释当前规则。
2. 请求总结当前争点。
3. 请求暂停或恢复。
4. 举报刷屏、跑题或不当内容。
5. 请求触发气氛效果。
6. 请求 NPC 评价某段公开发言是否精彩。

呼叫结果：

1. NPC 可以回应。
2. NPC 可以拒绝，并说明原因。
3. NPC 可以进入观察，不立即行动。
4. NPC 可以把问题交给运营或管理员处理。

交互边界：

1. NPC 只在房间公开上下文中回应。
2. NPC 不提供私聊窗口、私密问答或个人专属辩论建议。
3. 用户不得通过公开呼叫让 NPC 代替自己攻击、羞辱或压制其他用户。

### 8.2 用户查看 NPC 最近行为

房间应提供入口，让用户查看 NPC 的近期关键行为。

至少包含：

1. 时间。
2. 行为类型。
3. 行为内容。
4. 简短原因。
5. 是否由人工接管或撤销。

### 8.3 用户反馈

用户可以对 NPC 行为给出轻量反馈。

反馈类型：

1. 有帮助。
2. 打断过多。
3. 不够中立。
4. 看不懂。
5. 其他问题。

反馈不应在房间内公开展示为对用户的争执内容。

---

## 9. 运营控制

### 9.1 启用策略

运营应能控制 NPC 是否在某类场次启用。

可配置范围：

1. 全局默认开启或关闭。
2. 按辩题、赛事、房间类型开启或关闭。
3. 按场次临时开启或关闭。
4. 高风险活动中限制部分动作。

### 9.2 风格配置

运营可选择 NPC 的表达风格。

建议首版风格：

1. 娱乐裁判：活跃、会赞赏精彩发言、强调现场感。
2. 标准裁判：中立、克制、规则导向。
3. 活跃主持：更强节奏提示和气氛调动。
4. 安静观察：只在必要场景发言。

风格配置可以影响表达活跃度、赞赏频率和动效强度，但不得改变 NPC 的公平、中立和正式裁决隔离原则。

### 9.3 能力开关

运营应能控制以下能力：

1. 是否允许 NPC 主动发言。
2. 是否允许 NPC 点名赞赏用户公开发言。
3. 是否允许 NPC 暂停辩论。
4. 是否允许 NPC 触发特效。
5. 是否允许 NPC 进行阶段总结。
6. 是否允许用户公开呼叫 NPC。
7. 是否允许 NPC 对用户行为做公开警告。

### 9.4 人工接管

运营或管理员可在必要时接管 NPC 行为。

接管能力包括：

1. 暂停 NPC 主动行为。
2. 代表 NPC 发布房间公告。
3. 撤销或补充解释某次 NPC 行为。
4. 临时关闭强动作能力。
5. 将房间恢复为无 NPC 模式。

---

## 10. 关键业务规则

### 10.1 中立规则

1. NPC 不能提前宣布某一方获胜。
2. NPC 不能基于用户身份、等级、消费、历史胜率表达差异化态度。
3. NPC 对类似行为的提醒标准应一致。
4. NPC 公开总结应以内容为中心，不以用户人格为中心。
5. NPC 可以赞赏具体用户的公开发言，但赞赏依据必须是该发言的内容质量、表达效果或现场价值。
6. NPC 不需要机械平均赞赏双方，但长期赞赏分布若显著倾斜，应能从公开发言质量和房间上下文解释。

### 10.2 娱乐表现规则

1. NPC 可以使用更活泼的语气制造现场感。
2. NPC 可以对精彩发言、幽默表达、关键反击触发赞赏或动效。
3. NPC 的调侃不能针对用户身份、外貌、地域、性别、消费能力或历史表现。
4. NPC 的娱乐表达不能削弱正式规则、房间秩序或裁决边界。
5. NPC 不能用娱乐口吻包装胜负预测或阵营评分。

### 10.3 发言频率规则

1. NPC 主动发言必须有频率上限。
2. 低价值重复提示应被抑制。
3. 当用户密集发言时，NPC 应优先避免打断关键表达。
4. 重要动作优先于普通闲聊式回应。
5. 赞赏消息也应受频率控制，避免用户为了触发赞赏而刷屏。

### 10.4 强动作规则

强动作包括暂停、恢复、警告升级、特效覆盖、阶段切换提示。

规则要求：

1. 强动作必须有明确原因。
2. 强动作必须在房间内留下可见记录。
3. 强动作不能绕过房间既有规则。
4. 用户应能知道强动作由 NPC 触发还是人工触发。

### 10.5 正式裁决隔离规则

1. NPC 的发言不得被包装成最终判决。
2. NPC 不公开展示正式裁判团内部推理、置信度或审计信息。
3. NPC 阶段总结不应直接进入最终裁决报告作为唯一依据。
4. 正式裁决页应清楚标明其来自赛后官方裁判流程，而不是来自赛中 NPC。
5. NPC 永远不会替代 AI 裁判团生成正式裁决报告。
6. NPC 对用户发言的赞赏不进入正式裁决评分口径。

---

## 11. 异常与降级体验

### 11.1 NPC 不可用

当 NPC 不可用时：

1. 房间应继续可正常辩论。
2. 用户应看到轻量状态提示。
3. 不应影响用户发送消息、观战或最终裁决。
4. 正式 AI 裁判团流程不应因 NPC 不可用而被用户误认为失败。

### 11.2 NPC 行为争议

当用户认为 NPC 行为不公平时：

1. 用户可提交反馈。
2. 运营可查看该次行为记录。
3. 运营可撤销或补充解释现场行为。
4. 该争议不自动等同于正式裁决 challenge。

### 11.3 房间高风险内容

当房间出现明显违规、攻击性内容或恶意刷屏时：

1. NPC 可以先做规则提醒。
2. 必要时可触发暂停或升级给管理员。
3. 用户应看到当前房间进入秩序处理状态。
4. 处理完成后应恢复清晰的辩论状态。

---

## 12. 验收标准

### 12.1 基础展示

1. 启用 NPC 的房间中，用户能明确看到 NPC 身份和动态形象。
2. NPC 消息与普通用户消息视觉上可区分。
3. 用户能看到 NPC 当前状态。
4. 用户能通过视觉样式明确区分 NPC 与赛后官方裁决入口。
5. NPC 不可用时，房间核心辩论功能不受影响。

### 12.2 行为能力

1. NPC 能发布开场、阶段、规则、争点、节奏类公开消息。
2. NPC 能对精彩发言、高质量反驳、幽默表达或关键追问做出公开赞赏。
3. NPC 能在配置允许时触发暂停和恢复。
4. 暂停、恢复、警告等强动作均有原因说明。
5. NPC 能触发不遮挡核心操作的房间特效和动态形象反馈。
6. NPC 能生成不包含最终胜负判断的阶段总结。

### 12.3 运营控制

1. 运营可按场次开启或关闭 NPC。
2. 运营可调整 NPC 风格。
3. 运营可关闭高风险能力。
4. 运营可控制赞赏、动效、公开呼叫等娱乐能力。
5. 运营可人工接管或暂停 NPC 主动行为。

### 12.4 边界保护

1. NPC 不直接宣布最终胜负。
2. NPC 不公开阵营评分。
3. NPC 不暴露正式裁判团内部审计字段。
4. NPC 永远不会替代 AI 裁判团生成正式裁决报告。
5. NPC 不提供私聊入口。
6. NPC 对用户发言的赞赏不构成阵营优势、官方评分或裁决依据。
7. 用户能区分 NPC 现场行为与赛后官方裁决。

---

## 13. 非目标

长期明确不做：

1. NPC 替代 AI 裁判团生成正式裁决报告。
2. NPC 作为用户私人辩论教练或私聊助手。
3. NPC 代用户发言或自动参赛。
4. 基于 NPC 现场判断进行即时胜负结算。
5. NPC 公开阵营评分、胜率预测或内部裁判依据。
6. NPC 因赞赏某个用户而影响正式裁决。
7. 用户自由编辑 NPC 行为逻辑。

首版暂不做：

1. 多角色 NPC 剧情系统。
2. 完整 AI 主持人直播间系统。
3. 用户自定义 NPC 皮肤或语气。

---

## 14. 后续演进方向

后续可评估：

1. 不同赛事风格的裁判 NPC。
2. 观战用户可触发的有限互动效果。
3. 房间主题化 NPC 皮肤。
4. 大型赛事中的主持人模式。
5. 更丰富的动态形象、赞赏动作和房间气氛特效。
6. 赛后报告中引用 NPC 标记过的公开关键节点，但不把 NPC 赞赏当作裁决依据。
7. 面向运营的 NPC 体验分析面板。
