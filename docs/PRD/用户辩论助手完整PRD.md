# 用户辩论助手完整 PRD（v0.1）

更新时间：2026-05-01  
文档状态：新功能 PRD 草案  
适用范围：会员用户在辩论过程中可使用的私人辩论助手能力

---

## 0. 文档目的

本文档用于定义 EchoIsle 的“用户辩论助手”功能，明确：

1. 用户辩论助手的产品定位、会员价值、使用边界与公平约束。
2. 助手在辩论过程中能回答什么、能总结什么、能提供什么建议。
3. 助手与虚拟裁判 NPC、赛后 AI 裁判团、公开房间消息之间的区别。
4. 首版功能范围、交互流程、权限规则、验收标准与非目标。

说明：

1. 本文档只描述产品功能细节，不展开模型、知识库、检索、Agent 编排或服务架构。
2. 用户辩论助手是会员专属私人助手，不是公开裁判，也不代表平台正式观点。
3. 助手可以帮助用户理解辩论、整理思路和准备表达，但不得替用户自动参赛或绕过平台公平规则。

## 1. 功能定位

### 1.1 功能定义

用户辩论助手是会员用户在 Debate Room 内可打开的私人 AI 助手。

它具有当前辩论状态的上下文理解能力，能够帮助用户：

1. 了解当前场次、辩题、双方立场和进展。
2. 快速总结正反方观点、争点、未回应问题和关键发言。
3. 查询当前辩论相关的公开信息。
4. 获得发言建议、反驳思路、表达结构和赛后复盘帮助。
5. 降低参与辩论的门槛，提升会员用户的参与质量和付费价值感。

### 1.2 与其他 AI 角色的区别

| 维度 | 用户辩论助手 | 虚拟裁判 NPC | AI 裁判团 |
|---|---|---|---|
| 可见性 | 仅用户本人可见 | 房间公开可见 | 赛后结果公开可见 |
| 主要目标 | 私人理解、总结、建议 | 房间互动、秩序、节奏 | 官方裁决、评分、报告 |
| 面向对象 | 单个会员用户 | 房间内所有用户 | 整场辩论 |
| 是否代表平台裁决 | 否 | 否 | 是 |
| 是否能影响房间状态 | 默认不能 | 可在授权范围内影响 | 通过正式裁决影响终局 |
| 是否可给发言建议 | 可以，受公平规则约束 | 不提供私人建议 | 不提供赛中建议 |

### 1.3 核心价值

1. 提升会员权益感，让付费用户获得明确的辩论辅助能力。
2. 降低新用户参与门槛，使用户更容易理解复杂辩论。
3. 帮助参赛用户更好地组织表达，提高辩论质量。
4. 帮助观战会员更快看懂局势，提升观战粘性。
5. 为平台形成“赛中辅助 + 赛后裁决 + 长期复盘”的 AI 产品矩阵。

## 2. 用户角色

### 2.1 会员参赛用户

会员参赛用户正在正方或反方参与辩论，主要需求包括：

1. 快速理解对方刚刚提出的论点。
2. 找到自己阵营还没有回应的问题。
3. 获取下一条发言的结构建议。
4. 在时间有限时获得简洁、可直接参考的反驳思路。
5. 赛后复盘自己阵营表现。

### 2.2 会员观战用户

会员观战用户不参与发言，主要需求包括：

1. 看懂当前双方到底在争什么。
2. 查询某个术语、规则或背景信息。
3. 总结目前谁在什么维度上更有优势，但不把它当作正式裁决。
4. 获得观战视角的精简解说。

### 2.3 非会员用户

非会员用户可以看到助手入口或权益提示，但不能完整使用会员助手能力。

非会员体验目标：

1. 清楚知道该能力属于会员权益。
2. 能看到助手可以解决什么问题。
3. 可通过购买或升级会员解锁。
4. 不因未开通会员而影响基础辩论、观战和正式裁决体验。

## 3. 产品目标

### 3.1 核心目标

1. 让会员用户在辩论过程中获得高价值、低打扰的私人 AI 辅助。
2. 帮助用户更快理解当前辩论状态和关键争点。
3. 提升参赛用户发言质量，但不让助手变成代打工具。
4. 形成可感知、可复用、可持续扩展的会员权益。

### 3.2 成功指标

1. 会员助手入口点击率。
2. 会员助手使用后发言率提升。
3. 会员用户场次留存时长提升。
4. 助手回答满意度。
5. 助手使用后会员转化率或续费率提升。
6. 关于“代打、不公平、误导”的用户投诉率保持在低位。

## 4. 功能范围

### 4.1 首版包含

1. 会员用户可在辩论房间内打开私人助手面板。
2. 助手可回答当前辩论状态相关问题。
3. 助手可总结双方观点、关键争点、未回应问题和最新进展。
4. 助手可提供发言建议、反驳方向、表达结构和注意事项。
5. 助手可生成用户私有的赛中笔记或复盘摘要。
6. 非会员用户可看到入口和会员权益说明。

### 4.2 首版不包含

1. 助手代替用户发送房间消息。
2. 助手自动持续监听并弹出大量主动建议。
3. 助手公开影响房间状态、暂停辩论或触发特效。
4. 助手查看其他用户的私密信息。
5. 助手提供正式胜负预测或公开裁决。
6. 多助手协同、队伍共享助手或战队教练席。

## 5. 核心体验原则

### 5.1 私密性

助手是用户个人能力，默认只对当前用户可见。

产品要求：

1. 助手对话不进入公开房间消息流。
2. 其他用户不能看到该用户问了什么、助手答了什么。
3. 用户可自行决定是否把助手建议改写后发到房间。
4. 助手不得泄露其他用户未公开的信息。

### 5.2 辅助而非代打

助手应增强用户表达能力，而不是替用户完成辩论。

产品要求：

1. 助手可以给结构、方向、提醒和示例。
2. 助手不应一键代发完整发言。
3. 助手生成的建议应鼓励用户理解和改写。
4. 助手不应自动连续给出“下一句该怎么说”的强操控式指导。

### 5.3 与正式裁决隔离

助手可以分析局势，但不能伪装成正式裁判。

产品要求：

1. 助手不得宣称“本场最终会由某方获胜”。
2. 助手可用“当前公开内容看起来”表达临时分析。
3. 助手必须提示正式结果以赛后官方裁决为准。
4. 助手不得展示裁判团内部评分、置信度、审计信息或未公开依据。

### 5.4 低打扰

助手应由用户主动打开和使用，默认不干扰辩论主界面。

产品要求：

1. 助手入口清晰但不遮挡发言区。
2. 助手回复应短而可操作，支持展开查看。
3. 助手主动提醒应非常克制，仅用于高价值场景。
4. 用户可关闭助手面板或静音主动提醒。

## 6. 助手入口与会员权益

### 6.1 房间入口

Debate Room 中应提供助手入口。

对会员用户：

1. 可直接打开助手面板。
2. 面板应显示当前助手可用状态。
3. 面板应提供推荐问题或快捷操作。

对非会员用户：

1. 可看到锁定态入口。
2. 点击后展示会员权益说明。
3. 可进入会员开通路径。
4. 关闭后不影响房间基础使用。

### 6.2 会员权益说明

权益说明应突出真实使用价值。

建议文案方向：

1. 总结当前争点。
2. 找出对方论点漏洞。
3. 生成下一轮发言思路。
4. 回顾自己阵营表现。
5. 看懂复杂术语和规则。

权益说明不应承诺：

1. 保证获胜。
2. 自动替你发言。
3. 获得内部裁判答案。
4. 查看其他用户隐藏信息。

### 6.3 使用额度

产品应定义合理的会员使用边界。

可选维度：

1. 每场可提问次数。
2. 每日可用次数。
3. 回复长度或高级能力限制。
4. 免费体验次数。

额度限制应清晰展示，避免用户在关键时刻突然不知道为何不可用。

## 7. 问答能力

### 7.1 当前状态问答

用户可以询问当前辩论状态。

可回答问题示例：

1. 现在辩论到哪个阶段了？
2. 当前辩题是什么？
3. 正方和反方分别是什么立场？
4. 最近双方主要在争什么？
5. 刚才对方的核心观点是什么？
6. 我方还有哪些问题没回应？

回答要求：

1. 基于当前公开辩论内容。
2. 清楚标明不确定或信息不足的部分。
3. 优先给出简洁结论，再展开解释。
4. 不混入正式裁决口径。

### 7.2 规则与背景问答

用户可以询问与当前辩论相关的规则和背景。

可回答问题示例：

1. 这个辩题里的某个术语是什么意思？
2. 当前游戏版本里某个机制如何影响论点？
3. 这条规则对双方论证有什么影响？
4. 为什么这条证据重要？

回答要求：

1. 区分事实、解释和建议。
2. 对版本相关内容给出明确上下文。
3. 不把不确定背景包装成确定事实。
4. 不提供违反平台规则或赛事规则的建议。

### 7.3 裁决报告相关问答

在赛后或判决中状态下，助手可以帮助用户理解公开裁决内容。

可回答问题示例：

1. 裁决报告的核心理由是什么？
2. 我方为什么输了？
3. 哪些发言被报告重点引用？
4. 下次应该改进哪些维度？

回答要求：

1. 仅基于用户可见的公开报告字段。
2. 不暴露内部裁判审计字段。
3. 不宣称可以修改正式结果。
4. 对复核、draw、二番战等流程给出清晰说明。

## 8. 总结能力

### 8.1 全局总结

助手可总结当前整场辩论。

总结内容包括：

1. 辩题与双方立场。
2. 当前主要争点。
3. 双方核心论点。
4. 关键转折发言。
5. 尚未回应的问题。
6. 下一步值得关注的方向。

### 8.2 阵营总结

参赛用户可请求助手从自己阵营视角总结。

总结内容包括：

1. 我方已经建立的主要论点。
2. 我方论证中薄弱或重复的部分。
3. 对方尚未回应我方的问题。
4. 我方需要优先回应的对方攻击点。
5. 下一轮发言的建议优先级。

### 8.3 对方观点总结

用户可请求助手总结对方观点。

总结内容包括：

1. 对方最新核心论点。
2. 对方使用的主要证据或例子。
3. 对方论证链条。
4. 可能存在的漏洞或需要追问之处。

### 8.4 赛后复盘总结

辩论结束后，助手可生成私有复盘。

复盘内容包括：

1. 用户所在阵营表现。
2. 关键失分或优势点。
3. 可复用的优秀表达。
4. 下次辩论改进建议。
5. 与正式裁决报告的对应关系。

---

## 9. 建议能力

### 9.1 发言结构建议

助手可以帮助用户组织下一条发言。

建议形式：

1. 先回应对方核心点。
2. 再补充我方证据。
3. 最后抛出追问或总结。

输出要求：

1. 给结构，不直接代发。
2. 鼓励用户用自己的语言表达。
3. 控制长度，适合当前房间节奏。
4. 避免人身攻击、嘲讽或违规表达。

### 9.2 反驳方向建议

助手可以提出可选择的反驳角度。

建议内容包括：

1. 概念边界反驳。
2. 证据可靠性反驳。
3. 因果链条反驳。
4. 规则适用反驳。
5. 对方未回应问题追问。

输出要求：

1. 明确每个方向适合解决什么问题。
2. 不保证某方向一定能赢。
3. 不捏造证据。
4. 对不确定信息给出提示。

### 9.3 表达优化建议

助手可以帮助用户改进草稿。

可支持：

1. 更清晰。
2. 更短。
3. 更有逻辑。
4. 更适合当前辩论语气。
5. 去除攻击性表达。

限制：

1. 不应把草稿变成平台替用户完成的完整代打稿。
2. 不应加入用户没有提供、且当前公开内容无法支撑的新事实。
3. 不应生成骚扰、辱骂、歧视或恶意操控内容。

### 9.4 学习型建议

助手可给用户长期能力提升建议。

建议内容包括：

1. 你的论点经常缺少什么。
2. 你的反驳哪里最有效。
3. 你下次可以练习哪类表达。
4. 如何更好地利用证据和追问。

首版可只提供单场内建议，不强制形成长期用户画像。

## 10. 主动提醒

### 10.1 低频主动提醒

助手可在少量高价值场景主动提醒用户。

可提醒场景：

1. 对方提出了直接攻击我方核心论点的新观点。
2. 我方长时间没有回应某个关键问题。
3. 当前阶段即将结束。
4. 用户正在输入的草稿可能包含违规表达。

主动提醒要求：

1. 默认低频。
2. 用户可关闭。
3. 不遮挡主输入区。
4. 不直接替用户决定下一步。

### 10.2 禁止过度干预

助手不应：

1. 每条对方消息后都弹出建议。
2. 用强提醒迫使用户按助手思路发言。
3. 在用户未打开助手时长篇输出。
4. 把临时局势分析包装成胜负预测。

---

## 11. 公平与合规边界

### 11.1 允许的帮助

助手可以：

1. 总结公开辩论内容。
2. 解释公开规则和背景。
3. 帮助用户组织表达。
4. 提醒未回应争点。
5. 优化用户提供的草稿。
6. 赛后辅助理解裁决报告。

### 11.2 不允许的帮助

助手不得：

1. 自动替用户发送消息。
2. 承诺胜负结果。
3. 提供内部裁判结论、置信度或审计信息。
4. 泄露其他用户私密数据。
5. 生成违规、攻击、骚扰或歧视内容。
6. 伪造证据或引用不存在的背景信息。
7. 绕过会员限制、房间规则或赛事规则。

### 11.3 付费公平说明

由于助手是会员权益，产品需避免让用户认为付费等于官方偏袒。

规则要求：

1. 助手只影响用户个人理解和表达，不影响正式裁决标准。
2. AI 裁判团不因用户是否开通会员而改变评分标准。
3. 正式裁决不展示或使用会员身份作为判断依据。
4. 非会员仍可正常参与辩论、观战和获得官方裁决。

## 12. 典型用户流程

### 12.1 会员参赛用户获取下一轮建议

1. 用户进入辩论房间并选择阵营。
2. 用户打开助手面板。
3. 用户点击“帮我总结对方刚才的观点”。
4. 助手返回对方核心观点和漏洞。
5. 用户点击“给我下一轮发言结构”。
6. 助手给出简短结构建议。
7. 用户自行组织语言并发送到房间。

### 12.2 会员观战用户理解局势

1. 用户进入进行中场次观战。
2. 用户打开助手面板。
3. 用户询问“现在双方主要争点是什么？”
4. 助手给出当前争点、双方立场和最新进展。
5. 用户继续观看或跳转查看关键消息。

### 12.3 非会员用户看到权益

1. 非会员进入辩论房间。
2. 用户点击助手入口。
3. 页面展示会员助手可提供的能力。
4. 用户可选择开通会员或关闭入口。
5. 关闭后继续正常辩论或观战。

### 12.4 赛后复盘

1. 辩论结束并生成官方裁决报告。
2. 会员用户打开助手。
3. 用户询问“我方为什么输 / 赢 / 平局？”
4. 助手基于公开报告和房间内容进行解释。
5. 用户获得下次可改进的表达建议。

## 13. 界面与交互要求

### 13.1 助手面板

助手面板应支持：

1. 当前对话区。
2. 快捷问题。
3. 当前场次上下文提示。
4. 额度或会员状态提示。
5. 关闭、收起和清空对话。

### 13.2 快捷问题

建议首版快捷问题：

1. 总结当前争点。
2. 对方刚才说了什么？
3. 我方还没回应什么？
4. 给我一个发言结构。
5. 帮我优化这段草稿。
6. 解释当前裁决报告。

### 13.3 回复呈现

助手回复应支持：

1. 简短结论。
2. 要点列表。
3. 可展开详情。
4. 明确的不确定提示。
5. 必要的免责声明：正式结果以官方裁决为准。

### 13.4 草稿辅助

当用户使用草稿辅助时：

1. 用户输入草稿。
2. 助手给出优化建议或多个表达版本。
3. 用户自行选择、修改和发送。
4. 助手不应直接写入公开房间消息流并自动发送。

## 14. 异常与降级体验

### 14.1 助手不可用

当助手不可用时：

1. 用户应看到明确状态。
2. 房间核心辩论和观战不受影响。
3. 若用户为会员，应提示稍后重试或查看权益状态。
4. 不应影响正式裁决流程。

### 14.2 回答不确定

当助手缺少足够上下文时：

1. 应明确说明当前无法确定。
2. 可建议用户查看具体消息或补充问题。
3. 不应编造事实。
4. 不应给出过度确定的判断。

### 14.3 额度耗尽

当用户额度耗尽时：

1. 应展示剩余额度为 0。
2. 应说明恢复时间或升级方式。
3. 已有对话可继续查看。
4. 不应突然清空用户上下文。

---

## 15. 验收标准

### 15.1 会员入口

1. 会员用户可在房间内打开助手。
2. 非会员用户点击入口会看到权益说明和开通路径。
3. 助手面板可关闭和收起，不影响主辩论界面。
4. 用户能看到助手当前可用状态。

### 15.2 问答与总结

1. 助手能回答当前辩题、阶段、双方立场等状态问题。
2. 助手能总结双方主要观点和当前争点。
3. 助手能指出用户所在阵营未回应的问题。
4. 助手在信息不足时能明确说明不确定。

### 15.3 建议能力

1. 助手能提供下一轮发言结构。
2. 助手能提供反驳方向。
3. 助手能优化用户草稿。
4. 助手不会自动发送公开房间消息。

### 15.4 边界保护

1. 助手不会宣称最终胜负。
2. 助手不会暴露裁判团内部字段。
3. 助手不会查看或泄露其他用户私密信息。
4. 助手不会生成违反房间规则的建议。
5. 正式裁决不因用户是否使用助手而改变标准。

---

## 16. 非目标

首版明确不做：

1. 自动代打。
2. 一键代发。
3. 公开教练席。
4. 队伍共享助手。
5. 基于助手建议的胜负预测。
6. 访问裁判团内部推理。
7. 长期个人能力画像和跨场训练计划。
8. 让助手影响房间状态、暂停辩论或触发特效。

---

## 17. 后续演进方向

后续可评估：

1. 更细分的会员等级和助手能力。
2. 单场私人复盘档案。
3. 跨场个人表达成长报告。
4. 团队赛中的共享教练模式。
5. 观战解说模式。
6. 与正式裁决报告联动的学习建议。
7. 可选择的助手语气和辅助强度。
