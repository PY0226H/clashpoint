## 风险与回退

1. 风险：新增 `expectedUpdatedAt` 可能影响旧客户端更新成功率。
   - 回退：字段为可选；旧客户端不传时行为保持兼容。
2. 风险：新增审计 action 需要 DB 约束迁移。
   - 回退：migration 可单独回滚；不影响主表读写。
3. 风险：lock timeout 可能引入更多 409。
   - 回退：可调高超时阈值或临时关闭映射逻辑。
