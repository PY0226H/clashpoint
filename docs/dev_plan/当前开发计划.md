# API057 开发计划（slot: default）

更新时间：2026-04-08
对应文档：`docs/dev_plan/当前开发文档.md`
目标接口：`GET /api/debate/ops/rbac/me`

---

## 1. 目标与范围

本轮聚焦 P0/P1 的可落地项，并执行到可验证状态：

1. 统一 Owner 真源，移除 `owner_id=1` 硬编码依赖。
2. 补齐 `/api/debate/ops/rbac/me` OpenAPI 运行态错误契约。
3. 增加 route 级回归测试，覆盖 `200/401/403` 主路径。

不在本轮范围：

1. 多 owner / 多租户能力。
2. 全平台结构化错误模型统一改造。
3. 完整 observability 指标体系重构。

---

## 2. 实施方案（执行顺序）

### Phase A：Owner 真源统一（P0）

1. 新增 migration：平台 Owner 真源表（单行模型）。
2. `rbac.rs`：`get_platform_admin_user_id()` 改为查库；Owner 缺失时返回 `ServerError`。
3. `rbac.rs`：`grant_platform_admin()` 同步写入 Owner 真源与 `platform_user_roles`。
4. `ops_observability.rs`：告警接收人 Owner 来源改为 Owner 真源表，避免继续硬编码 `id=1`。

### Phase B：契约对齐（P1）

1. `debate_ops.rs`：`get_ops_rbac_me_handler` OpenAPI responses 增加 `401/403/500`。

### Phase C：测试封板（P1）

1. `handlers/debate.rs` 增加 route 级测试：
   - 已登录已绑手机用户访问 `/api/debate/ops/rbac/me` 返回 `200`。
   - 缺失 token 返回 `401`。
   - 未绑手机返回 `403 auth_phone_bind_required`。
2. `models/rbac.rs` 增加 Owner 真源相关测试：验证 `grant_platform_admin()` 后 owner 快照语义。

### Phase D：验证与收尾

1. 运行针对性测试（`chat-server` 目标测试模块）。
2. 若通过，回填计划执行结果与风险备注。

---

## 3. 验收标准（DoD）

1. 运行时不再依赖 `get_platform_admin_user_id -> Ok(1)`。
2. `/api/debate/ops/rbac/me` OpenAPI 至少声明 `200/401/403/500`。
3. route 级测试新增并通过（200/401/403）。
4. Owner 真源变更后，`isOwner/roleManage` 行为与预期一致。

---

## 4. 风险与回滚

1. 风险：Owner 真源缺失会导致管理能力不可用。
2. 兜底：`grant_platform_admin()` 在赋权时自动写入 Owner 真源，确保 bootstrap 与测试链路可恢复。
3. 回滚：可回滚 migration + 恢复旧逻辑（仅在本地开发阶段使用）。
