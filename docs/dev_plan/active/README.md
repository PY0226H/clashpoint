# 活动计划目录

## 1. 用途

当需要并行推进多个短期开发计划时，推荐把活动计划文档放在本目录，并通过 `.codex/plan-slots/<slot>.txt` 绑定到对应线程。

## 2. 推荐命名

1. `backend-signin.md`
2. `frontend-ui.md`
3. `auth-session-hardening.md`

## 3. 使用规则

1. 一个线程只绑定一个 `slot`
2. 一个 `slot` 只对应一个活动计划文档
3. 收口后将已完成内容整合进 `completed.md`，未完成内容整合进 `todo.md`
4. 收口完成后，可清空、归档或重置对应活动计划文档
